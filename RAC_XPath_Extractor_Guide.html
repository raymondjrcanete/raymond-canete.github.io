Overview

RAC XPath Extractor is a Chrome browser extension built for QA automation engineers using Robot Framework. It automatically scans any web page and extracts XPaths for all interactive elements — outputting them directly as Robot Framework variable declarations, ready to paste into your test scripts.

💡 What it solves: Instead of manually inspecting each element in DevTools and writing XPaths one by one, this tool scans the entire page at once and generates all variables in the correct Robot Framework format automatically.

What It Extracts

Element TypeVariable PrefixXPath Strategy
ButtonsBTN_Uses button text: //button[normalize-space()='Text']
Text Input FieldsTXT_Uses placeholder, label, id, or name
TextareasTXT_Uses placeholder, label, or id
Dropdowns / SelectDDL_Uses id, label, or name
CheckboxesCHK_Uses id or name attribute
Radio ButtonsRDO_Uses id or name attribute
Popups / ModalsMSG_Uses role="dialog" or modal class

Installation

Step 1 — Get the Extension Files

The extension folder xpath-extractor-extension contains these files:

xpath-extractor-extension/
├── manifest.json    ← Extension configuration
├── content.js       ← Page scanning logic
├── popup.html       ← Extension UI
├── popup.js         ← UI logic
└── icon.png         ← Extension icon
Step 2 — Open Chrome Extensions
  1. Open Google Chrome
  2. Go to the address bar and type: chrome://extensions
  3. Press Enter
Step 3 — Enable Developer Mode
  1. Look for the "Developer mode" toggle in the top-right corner
  2. Click to turn it ON
  3. Three new buttons will appear: Load unpacked, Pack extension, Update
Step 4 — Load the Extension
  1. Click "Load unpacked"
  2. Browse to and select the xpath-extractor-extension folder
  3. Click Select Folder
  4. The RAC XPath Extractor extension will appear in your extensions list
Step 5 — Pin to Toolbar
  1. Click the puzzle piece icon (🧩) in the Chrome toolbar
  2. Find RAC XPath Extractor
  3. Click the pin icon 📌 to pin it to the toolbar
  4. The 🔍 icon will now appear in your Chrome toolbar for easy access
✅ Installation complete! The extension is now ready to use on any website.

How to Use — Step by Step

Step 1 — Navigate to the Page

Open the web application page you want to automate in Chrome. For example, a login page, transaction form, or any page with interactive elements.

Step 2 — Open the Extension

Click the 🔍 RAC XPath Extractor icon in the Chrome toolbar. The extension popup will open.

Step 3 — Set the Section Name

Type a meaningful section name in the 📁 Section Name field. This becomes the comment header in your output.

PageSuggested Section Name
Login pageLogin Page
Transaction formTransaction Form Fields
Customer registrationCustomer Registration Form
Search pageSearch Page Elements
Step 4 — Select Filters (Optional)

Toggle the filter chips to include or exclude specific element types. All are enabled by default. Click a chip to deactivate it (turns white) or activate it (turns dark blue).

Step 5 — Click Extract XPaths

Click the 🔍 Extract XPaths button. The tool will scan the entire page and display the results in the output area.

The status bar will show: ✅ Found X elements with count badges for each type.

Step 6 — Copy or Download
  • Click 📋 Copy All → paste directly into your .robot file
  • Click 💾 Download .robot → saves as a .robot file automatically named after your section

Output Format

The output is formatted as a Robot Framework *** Variables *** section:

*** Variables ***
# Transaction Form Fields

# 🔘 Buttons
${BTN_CREATE_NEW_CUSTOMER}              //button[normalize-space()='Create New Customer']
${BTN_QUICK}                            //button[normalize-space()='Quick']
${BTN_FULL}                             //button[normalize-space()='Full']

# 📝 Text Fields
${TXT_JUAN}                             //input[@placeholder='Juan']
${TXT_EMAIL}                            //input[@placeholder='Email Address']
${TXT_PASSWORD}                         //input[@type='password']

# 📋 Dropdowns
${DDL_STATUS}                           //select[@id='status']

# ☑️ Checkboxes
${CHK_AGREE}                            //input[@id='agree']
✅ Variables are automatically padded to 40 characters for clean alignment — matching Robot Framework coding standards.

Variable Naming Convention

The tool automatically generates variable names based on element attributes in this priority order:

For Buttons

SourceExample InputVariable Name
Button textCreate New Customer${BTN_CREATE_NEW_CUSTOMER}
Input valueSubmit${BTN_SUBMIT}
ID attributeid="btnSave"${BTN_BTNSAVE}
FallbackNo text/id${BTN_BUTTON_1}

For Text Fields

PrioritySourceExampleVariable Name
1stAssociated label<label>First Name</label>${TXT_FIRST_NAME}
2ndPlaceholderplaceholder="Juan"${TXT_JUAN}
3rdID attributeid="firstName"${TXT_FIRSTNAME}
4thName attributename="first_name"${TXT_FIRST_NAME}
5thFallbackNo attributes${TXT_FIELD_1}
💡 Duplicate names are automatically handled — if two elements generate the same name, the second one gets a _2 suffix (e.g. ${BTN_SUBMIT_2}).

Full Example

Scenario: Extracting from a Transaction Form

You are automating a transaction form with the following elements:

  • A "Create New Customer" button
  • A "Quick" button and a "Full" button
  • A text field with placeholder "Juan" (First Name)
  • A text field with placeholder "Email Address"
  • A dropdown for Status

Steps

  1. Navigate to the transaction form page in Chrome
  2. Click the 🔍 RAC XPath Extractor icon
  3. Set Section Name to: Transaction Form Fields
  4. Click Extract XPaths
  5. Click 📋 Copy All
  6. Paste into your Robot Framework .robot file

Tips & Best Practices

TipDetail
✅ Extract per page/formUse a different Section Name for each page or form to keep variables organized
✅ Capture popups while openTrigger the popup first, then click Extract to capture its elements
✅ Use Download for large pagesFor pages with many elements, download as .robot file instead of copying
✅ Review before usingAlways review generated XPaths — some may need manual adjustment for dynamic pages
✅ Rename vague variablesIf a variable is named ${TXT_FIELD_1}, rename it to something meaningful
⚠️ Dynamic contentXPaths based on text may break if the page text changes — use ID-based XPaths when available
⚠️ iFramesElements inside iFrames may not be captured — switch to the iFrame context first
⚠️ Single Page AppsFor React/Angular/Vue apps, wait for the page to fully load before extracting
