Overview

The PGC Format Test Case Generator is a Python command-line tool that converts use case scenarios written in Given/When/Then format into a formatted Excel (.xlsx) test case file — matching the standard PGC QA template.

💡 What it does: You paste your use case scenarios → the tool automatically extracts each scenario → generates test case rows in Excel with proper formatting, TC IDs, test steps, and expected results.

Key Features

FeatureDescription
Auto-parses scenariosDetects Scenario 1, Scenario 2... automatically
Extracts user roleReads "Persona:" or "As a..." and fills Test Data column
Builds test stepsFormats Given/When/Then into structured test steps
Generates expected resultsConverts THEN clause into expected result
Auto TC ID formulaExcel formula auto-generates TC IDs when ticket is entered
Append modeAdd more scenarios to an existing file without overwriting
Color-coded ExcelAmber header, blue section rows, white data rows

Requirements & Installation

Prerequisites

  • Python 3.10 or higher
  • openpyxl library

Install Dependency

Open Command Prompt and run:

pip install openpyxl

Verify Python Version

python --version

Expected output: Python 3.10.x or higher

How to Run

  1. Open Command Prompt
  2. Navigate to the tool folder:
cd "D:\QA Tools\Others\TescaseGenerator"
  1. Run the tool:
python tc_generator.py
⚠️ Do NOT double-click the .py file — it may open a text editor instead of running. Always use python tc_generator.py in Command Prompt.

Scenario Format

The tool expects scenarios in this structure:

Persona: Rematado Processing Clerk
Feature: Settings > Price and Rate

As a user, I want to...

Scenario 1: Title of the scenario
GIVEN: some precondition
WHEN: some action
THEN: expected outcome
AND: additional condition (optional)

Scenario 2: Another scenario
GIVEN: another precondition
WHEN: another action
THEN: another expected outcome

Key Rules

RuleDetail
Scenario headersMust start with Scenario N: (e.g. Scenario 1:, Scenario 2:)
KeywordsGIVEN, WHEN, THEN, AND — case insensitive
AND clausesAutomatically attached to the preceding keyword (GIVEN/WHEN/THEN)
SAMPLE UI sectionsAutomatically ignored and stripped from output
Persona lineOptional — auto-fills Test Data column if present

Full Example Session

Input Scenario

As a Requestor,
I WANT to submit a User Enrollment request.

Scenario 1: Displaying the Request for someone else option
GIVEN: The Requestor is within the Online Banking Request Form.
WHEN: The request form is displayed.
THEN: The system must show a checkbox labeled Request for someone else.
AND: The checkbox must be Unchecked by default.

Scenario 2: Behavior if checkbox is checked
GIVEN: The Requestor is viewing the User Enrollment request form.
WHEN: The Requestor checks the Request for someone else checkbox.
THEN: The system must hide the Data Privacy Notice.
AND: Set all applicable fields to uneditable.

Tips & Best Practices

TipDetail
✅ Always include Persona or As aEnsures Test Data column is auto-filled correctly
✅ Use Append mode for same ticketKeeps all scenarios for one ticket in one file
✅ Use SAMPLE UI sections freelyTool automatically ignores them — no need to remove before pasting
✅ Leave Ticket blank if unknownTC ID formula will auto-fill once you enter the ticket in Excel
⚠️ Each scenario needs Scenario N:Without this header, the tool won't detect the scenario
⚠️ Don't forget to type ENDNothing processes until END is typed on its own line
⚠️ AND must follow a keywordAND without a preceding GIVEN/WHEN/THEN will be ignored
