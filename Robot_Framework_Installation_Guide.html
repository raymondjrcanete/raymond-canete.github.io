Prerequisites

Before starting, make sure you have the following:

  • Windows machine
  • Internet connection
  • Administrator access on your machine
Step 1 — Install Python

Robot Framework runs on Python, so this must be installed first.

  1. Go to https://www.python.org/downloads
  2. Click "Download Python 3.x.x" (latest stable version)
  3. Run the downloaded installer
  4. ⚠️ IMPORTANT: Check "Add Python to PATH" at the bottom of the installer screen before clicking Install. If you miss this, Python commands will not work in the terminal.
  5. Click "Install Now"
  6. Wait for installation to complete, then click Close

Verify Installation

Open Command Prompt (press Win + R, type cmd, press Enter) and run:

python --version

Expected output:

Python 3.x.x
pip --version

Expected output:

pip 23.x.x from ...
💡 If you see "python is not recognized", Python was not added to PATH. Reinstall and make sure to check "Add Python to PATH".
Step 2 — Set Up Virtual Environment

A virtual environment creates an isolated Python workspace for your project. This keeps your project's libraries separate from other projects and prevents version conflicts across your team.

💡 Why this matters: Without a virtual environment, all projects share the same Python libraries. If one project needs a different version of a library, it can break another project. A virtual environment prevents this.

Create the Virtual Environment

Open Command Prompt, navigate to your project folder, and run:

# Navigate to your project folder first
cd C:\path\to\my_project

# Create virtual environment (named "venv")
python -m venv venv

This creates a venv/ folder inside your project directory.

Activate the Virtual Environment

You must activate the virtual environment every time you work on the project:

OS Command
Windows (Command Prompt) venv\Scripts\activate
Windows (PowerShell) venv\Scripts\Activate.ps1
Mac / Linux source venv/bin/activate

Once activated, your terminal prompt will show (venv) at the beginning:

(venv) C:\my_project>
✅ When you see (venv) in your terminal, the virtual environment is active. All libraries you install will go into this isolated environment only.

Deactivate When Done

When you finish working, deactivate the virtual environment:

deactivate

Configure VS Code to Use the Virtual Environment

  1. Open your project folder in VS Code
  2. Press Ctrl + Shift + P
  3. Type "Python: Select Interpreter" and select it
  4. Choose the interpreter inside your venv folder:
    .\venv\Scripts\python.exe
💡 VS Code will automatically activate the virtual environment in its integrated terminal once the interpreter is selected.

Add venv to .gitignore

If your team uses Git, make sure to exclude the venv/ folder from version control. Create a .gitignore file in your project root and add:

venv/
results/
__pycache__/
*.pyc
⚠️ IMPORTANT: From this point forward, make sure your virtual environment is always activated before installing libraries or running tests. All pip install commands in the next steps should be run with (venv) active.
Step 3 — Install Robot Framework

Open Command Prompt and run the following command:

pip install robotframework

Verify Installation

robot --version

Expected output:

Robot Framework 7.x.x (Python 3.x.x)
Step 4 — Install Robot Framework Libraries

Install the most commonly used libraries for your team:

SeleniumLibrary (Web / UI Testing)

pip install robotframework-seleniumlibrary

RequestsLibrary (API Testing)

pip install robotframework-requests

DatabaseLibrary (Database Testing)

pip install robotframework-databaselibrary
💡 BuiltIn, Collections, and String libraries come pre-installed with Robot Framework — no additional installation needed.
Step 5 — Install Browser Driver (For UI Testing)

Required only if your team will perform web UI testing.

Option A — Manual ChromeDriver Installation

  1. Check your Chrome version: Chrome menu → Help → About Google Chrome
  2. Go to https://chromedriver.chromium.org/downloads
  3. Download the ChromeDriver version that matches your Chrome version
  4. Extract chromedriver.exe
  5. Place it in C:\Windows\System32 or add its folder to your system PATH

Option B — WebDriver Manager (Recommended / Easier)

pip install webdriver-manager
✅ WebDriver Manager automatically downloads and manages the correct driver version for you.
Step 6 — Install Visual Studio Code
  1. Go to https://code.visualstudio.com
  2. Click "Download for Windows"
  3. Run the installer
  4. Accept the license agreement
  5. Keep all default settings and click Next through all steps
  6. Check "Add to PATH" option if available
  7. Click Install
  8. Launch VS Code after installation completes
Step 7 — Install Robot Framework Extension in VS Code

This extension provides syntax highlighting, autocomplete, and test execution inside VS Code.

  1. Open VS Code
  2. Click the Extensions icon on the left sidebar (or press Ctrl + Shift + X)
  3. Search for "Robot Framework Language Server"
  4. Select the one published by Robocorp
  5. Click Install

Additional Recommended Extensions

Extension Name Publisher Purpose
Robot Framework Language Server Robocorp Syntax highlighting, autocomplete, test runner
Python Microsoft Python language support
Material Icon Theme Philipp Kief Better file icons for easier navigation
Step 8 — Configure VS Code for Robot Framework
  1. Open VS Code
  2. Press Ctrl + Shift + P to open the Command Palette
  3. Type "Open Settings (JSON)" and select it
  4. Add the following configuration:
{
    "robot.language-server.python": "python",
    "robot.python.executable": "python",
    "editor.tabSize": 4,
    "editor.insertSpaces": true,
    "files.associations": {
        "*.robot": "robotframework",
        "*.resource": "robotframework"
    }
}
  1. Save the file using Ctrl + S
Step 9 — Create Your First Project

Recommended Folder Structure

my_project/
│
├── tests/
│   └── sample_test.robot
│
├── resources/
│   └── keywords.resource
│
├── data/
│   └── test_data.csv
│
├── results/
│
└── requirements.txt

How to Create in VS Code

  1. Open VS Code
  2. Click File → Open Folder
  3. Create a new folder called my_project and open it
  4. Create the folders and files following the structure above
Step 10 — Write Your First Test

Create the file tests/sample_test.robot and add the following content:

*** Settings ***
Library    Collections

*** Variables ***
${NAME}    Robot Framework

*** Test Cases ***
First Test
    [Documentation]    This is my first Robot Framework test
    Log    Hello from ${NAME}!
    Should Be Equal    ${NAME}    Robot Framework

Second Test
    [Documentation]    Simple math check
    ${result}=    Evaluate    2 + 2
    Should Be Equal As Numbers    ${result}    4
💡 Robot Framework uses spaces (not tabs) to separate keywords and arguments. Use at least 2 spaces between keywords and their arguments.
Step 11 — Run Your First Test

Option A — From VS Code Terminal

  1. Open Terminal in VS Code using Ctrl + `
  2. Navigate to your project folder
  3. Run one of the following commands:
Command What it does
robot tests/ Run all tests in the tests folder
robot tests/sample_test.robot Run a specific test file
robot --outputdir results tests/ Run all tests and save results to results folder

Option B — From VS Code Directly

  1. Open the .robot file in VS Code
  2. You will see a ▶ Run button above each test case (provided by the Robot Framework extension)
  3. Click it to run that specific test
Step 12 — View Results

After running, Robot Framework automatically generates 3 output files:

File Description How to Open
output.xml Raw test results data Used by tools, not for manual reading
log.html Detailed step-by-step test log Open in any web browser
report.html Summary report with pass/fail status Open in any web browser

Run with Output Folder

robot --outputdir results tests/

Then open results/report.html in your browser to view the test report.

Step 13 — Create requirements.txt

This file allows your team to install all dependencies with a single command.

Create requirements.txt in your project root folder:

robotframework==7.0
robotframework-seleniumlibrary==6.3.0
robotframework-requests==0.9.7
robotframework-databaselibrary==1.4.3
webdriver-manager==4.0.1

Team Members Install All Dependencies With:

pip install -r requirements.txt
✅ This ensures all team members use the same library versions, avoiding compatibility issues.

Quick Verification Checklist

After completing all steps, run these commands to verify everything is working:

# Check Python
python --version

# Check pip
pip --version

# Check Robot Framework
robot --version

# Check SeleniumLibrary
python -c "import SeleniumLibrary; print('SeleniumLibrary OK')"

# Check RequestsLibrary
python -c "import RequestsLibrary; print('RequestsLibrary OK')"

All commands should return version numbers or "OK" without any errors.

Common Issues & Fixes

Problem Cause Fix
python not recognized Python not added to PATH Reinstall Python and check "Add Python to PATH"
robot not recognized Robot Framework not installed Run pip install robotframework
Extension not working in VS Code Wrong Python path in settings Set robot.language-server.python in VS Code settings
No autocomplete in .robot files Extension not configured properly Revisit Step 7 and verify settings JSON
ChromeDriver error Version mismatch with Chrome Match ChromeDriver version to your Chrome version or use webdriver-manager
IndentationError in test file Using tabs instead of spaces Use spaces only; set VS Code tab size to 4 spaces
Library not found error Library not installed Run pip install robotframework-[libraryname]
Libraries installed but not found Virtual environment not activated Run venv\Scripts\activate before installing or running tests
Wrong Python interpreter in VS Code VS Code not pointing to venv Press Ctrl+Shift+P → "Python: Select Interpreter" → choose venv\Scripts\python.exe

Folder Structure Best Practice

project/
│
├── venv/                   ← Virtual environment (DO NOT commit to Git)
│
├── tests/                  ← Test files (.robot)
│   ├── login_test.robot
│   ├── search_test.robot
│   └── api_test.robot
│
├── resources/              ← Reusable keywords (.resource)
│   ├── common.resource
│   └── api_keywords.resource
│
├── data/                   ← Test data files (.csv, .json)
│   └── users.csv
│
├── results/                ← Test results (auto-generated)
│
├── .gitignore              ← Excludes venv/ and results/ from Git
│
└── requirements.txt        ← Project dependencies
Folder Purpose File Types
tests/ Contains all test cases .robot
resources/ Reusable keywords shared across tests .resource
data/ Test data files used by tests .csv, .json, .xlsx
results/ Auto-generated test reports and logs .html, .xml
requirements.txt Lists all project dependencies .txt
